# FuseAI - AI Agent Generator

A modern React frontend for building, deploying, and monitoring AI agents that connect your favorite services.

## Features

- **Agent Dashboard** - View all your deployed agents with real-time status
- **Agent Creation Wizard** - Step-by-step wizard to create new agents from natural language prompts
- **Agent Monitoring** - View logs, metrics, and performance data
- **API Testing Playground** - Test your agent's API endpoints directly in the browser
- **Code Viewer** - View and download generated agent code

## Tech Stack

- **React 18** - Modern React with hooks
- **React Router** - Client-side routing
- **Lucide React** - Beautiful, consistent icons
- **CSS Variables** - Custom design system with minimalist color palette

## Getting Started

### Prerequisites

- Node.js 18+
- npm or yarn

### Installation

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

### Environment Variables

Create a `.env` file in the root directory:

```env
VITE_API_URL=http://localhost:8000/api
```

## Backend Integration

The frontend is designed to connect to a backend API. The API service layer (`src/services/`) provides a clean interface for all backend communication.

### API Endpoints Expected

```
GET    /api/agents           # List all agents
GET    /api/agents/:id       # Get agent details
POST   /api/agents           # Create new agent
POST   /api/agents/analyze   # Analyze prompt and generate plan
PATCH  /api/agents/:id       # Update agent
DELETE /api/agents/:id       # Delete agent
POST   /api/agents/:id/start # Start agent
POST   /api/agents/:id/stop  # Stop agent
POST   /api/agents/:id/restart # Restart agent
GET    /api/agents/:id/logs  # Get agent logs
GET    /api/agents/:id/metrics # Get agent metrics
POST   /api/agents/:id/test  # Test agent endpoint
GET    /api/agents/:id/code  # Get generated code
```

## Architecture

The project follows a clean architecture pattern:

1. **Presentation Layer** - React components and pages
2. **State Management** - React Context for global state
3. **Service Layer** - API abstraction for backend communication
4. **Data Layer** - Ready for integration with any REST API
